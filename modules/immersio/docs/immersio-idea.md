# immersio v0.1 — 중간고사 분석 워크플로우 설계

> 인체구조와기능(간호학과) 중간고사 결과와 사전진단평가 결과를 결합하여,
> "학생 맞춤형 관리"를 최우선 가치로 두고 분석을 운영하기 위한 설계 문서.

---

## 위상 (paideia 안에서의 위치)

본 문서는 **paideia v0.1의 첫 가동 모듈** `immersio`의 v0.1 명세이다.

- **paideia 우산 비전·로드맵**: `../../paideia-idea.md`
- **데이터 레이어 규약**: paideia 루트 `data/{bronze,silver,gold}/` (gitignore)
- **공유 스키마(Pydantic)**: `shared/paideia_shared/schemas/`
- **immersio가 paideia에서 맡는 역할**: 시험 결과를 needs-map의 학생 개인 데이터(사전진단)와 결합 분석 → 시험 품질 보고서 + 1인 1장 PDF 카드 + 라벨링된 상담 우선순위 명단 생성
- **v0.1 운영 목표**: 2026년 1학기 중간고사(184명)를 대상으로 본 워크플로우를 한 번 완주

## 0. 전제와 현재 진행 상황

### 0.1 데이터 자산 (paideia data/ 레이아웃 기준)

**Bronze (원시 입력)** — 각 데이터의 가변성 정도가 다름:
- `data/bronze/진단평가/진단평가_1차_결과.csv` **[가변]** — 응답 194명, 63문항. **과목마다 설문 문항이 달라짐** → immersio는 컬럼 의미를 설정파일(YAML)로 받고 코드에 컬럼명 하드코딩 금지. 코드는 매핑된 의미축(motivation/anxiety/interest 등)에만 의존.
- `data/bronze/시험성적/인체구조와기능_{A,B,C,D}반 결과(*).xls` **[고정]** — 학과 OMR 시스템 표준 출력. 분반별 결과·결시·OX·문항분석 4종 × 4분반 = 12개 + 백업 zip. **항상 같은 구조** → 단단한 파서 1개로 충분.
- `data/bronze/시험문제/{실제_출제문제,중간고사_문항,테스트용_문제}.yaml` **[고정 후보]** + 시험지(hwp/docx) + 채점키(xlsx) — yaml은 immersio가 정한 스키마(`paideia_shared.schemas.ExamItem`)로 표준화.
- `data/bronze/출석/출석부명단.xlsx` **[템플릿 제공]** (+ 분석사본) — 사용자가 온라인 출석부 시스템에서 복붙하여 수기 작성. **immersio가 표준 템플릿(`templates/attendance.xlsx`)을 제공**하고, 사용자는 그 템플릿에 데이터만 채워넣음.

**Silver (이전 LLM 분석 결과 — immersio가 새로 생성하여 대체할 것)**
- `data/silver/legacy/중간고사_분석결과.xlsx`
  - 시트: 전체요약 / 1_히스토그램 / 2_메타데이터통계 / 3_변별력 / 4_정답률 / 5_오답분석 / 학생성적

**Gold (이전 운영 산출물 — 참고용 baseline)**
- `data/gold/legacy/중간고사_분석요약.{md,pdf}`
- `data/gold/legacy/fig{1,2}_*.png`

**docs (영구 운영 문서, 모든 학기 공통)**
- `data/docs/상담/{상담신청_폼_문항,상담운영_가이드}.md`
- `data/docs/후속설문/중간고사_후속설문_{문항,익명형}.md`
- `data/docs/출제관리/{검수보고서,출제전략,형성평가_대조,교과서_근거_매핑}.md`

### 0.2 이미 완료된 분석
시험 자체에 대한 품질 점검(변별력·정답률·오답분석·메타데이터 통계·학생성적)은 LLM 도움을 받아 1차 완성 상태(`silver/legacy/`, `gold/legacy/`). 따라서 이번 immersio v0.1은 **"진단평가와 시험성적을 결합한 학생 맞춤형 분석"** 단계부터 본격 시작한다. 단, Phase 1(시험 품질) 자체도 코드로 재계산하여 **결정론적으로 재현 가능**하게 만든다.

### 0.3 핵심 설계 원칙
1. **수치·통계·매칭·룰 기반 분류는 전부 코드로 결정론적으로 처리** — 재현성·감사가능성 확보
2. **LLM은 "자연어를 다듬는 단계"에서만 선택적으로 사용** — 없어도 분석은 끝까지 완주
3. **LLM이 하던 일은 사전 정의된 룰 템플릿/키워드 사전으로 대체 가능**한 구조로 설계
4. 학생 맞춤형 산출물이 최종 목적 — 집단 통계는 그 보조
5. **모듈 간 데이터 계약은 `paideia_shared.schemas`의 Pydantic 모델로 명시** — Bronze 입력 검증, Silver/Gold 산출 검증

### 0.4 v0.1 산출 위치
- `data/silver/immersio/{학생마스터.parquet, 시험품질.parquet, 진단×시험.parquet, ...}`
- `data/gold/immersio/{cards/2026-1-anatomy/{학번}.pdf, 시험품질보고서.pdf, 라벨_상담우선순위.xlsx}`

### 0.5 입출력 계약 — 가변성에 따른 처리 전략

| 데이터 | 가변성 | immersio 처리 전략 |
|---|---|---|
| 진단평가 CSV | 과목마다 다름 | 컬럼 매핑 YAML(`config/{과목}.diagnostic.yaml`)을 별도 입력. 코드는 매핑된 의미축(motivation·anxiety·interest 등)에만 의존 |
| 시험성적 XLS | 학과 OMR 표준 | 단단한 파서 1개 (`immersio.io.exam_omr`). 시트명·컬럼 좌표 고정 가정 |
| 출석부 XLSX | 사용자 수기 | immersio가 표준 템플릿 제공 (`modules/immersio/templates/attendance.xlsx`). 사용자는 빈칸만 채움 |
| 시험문제 YAML | 사용자 작성 | `paideia_shared.schemas.ExamItem` Pydantic 검증. 위반 시 명확한 에러 |
| **분석결과** (silver/gold) | **immersio 출력** | **고정 스키마 + 고정 템플릿** — 매 학기 동일 구조의 xlsx/PDF가 나오도록. legacy의 7개 시트(전체요약·히스토그램·메타데이터통계·변별력·정답률·오답분석·학생성적)를 표준화 |

**핵심 원칙**: 입력의 가변성은 설정·템플릿으로 흡수하고, **출력은 항상 고정 구조**. 이래야 매 학기·다른 교과목에서 동일한 후속 운영(상담·회고)이 가능. 분석결과 xlsx/PDF의 시트명·컬럼·섹션 순서·차트 위치까지 명세하여 외부 도구(retro-mester 등)가 안전하게 소비.

## 2. 단계별 상세 — 코드 vs LLM

### Phase 0. 데이터 정합성·정제 — 전적으로 코드
- 진단평가 학번(2026XXX)과 시험성적 학번 매칭
- 결시 19명·미응답자 식별, 분반(A/B/C/D) 라벨 부여
- 진단의 텍스트 Likert ("매우 그렇다"~"매우 아니다") → 1–7 정수 매핑
- 다중선택 항목(주제 선호 등) 세미콜론 분해 → 원-핫 컬럼 30여 개로 펼치기
- 한 번 짜두면 1초 안에 재실행 가능. **LLM 불필요**.
- **출력 검증**: `paideia_shared.schemas.StudentMaster` Pydantic 모델로 행 단위 검증

### Phase 1. 시험 품질 분석 — 코드로 재현 (legacy 대체)
이미 LLM으로 1차 완성된 `silver/legacy/중간고사_분석결과.xlsx`를 코드로 재계산. 운영의 재현성을 위해 룰 기반 분류로 백업:

```
IF 최다오답률 > 30% AND 최다오답이 정답 인접 선택지 → "근접 distractor에 의한 변별 성공형"
IF 무응답률 > 10% → "시간 부족 또는 포기형"
IF 정답률 < 30% AND 변별력지수 > 0.3 → "어려운 변별 우수 문항(유지 권장)"
IF 변별력지수 < 0 → "역변별 의심 — 출제 재검토"
```

룰 기반은 LLM 산 문장보다 매끄럽지 않지만, **운영 측면에서 신뢰성·일관성 우수**. v0.3+ 시점에 py-irt 도입 시 IRT 모수 추정도 추가 (현재는 CTT).

### Phase 2. 학생별 성과 지표 — 전적으로 코드
학생 1명당 다음을 산출:
- 총점, 100점환산, 분반 내·전체 백분위, z-score
- **챕터별 정답률** (1장~13장 각각)
- **출처별 정답률** (교재 / 형성평가 / 퀴즈)
- **난이도별 정답률** (쉬움 / 중 / 어려움) + **예상난이도 vs 실제정답률 갭** (교수님 출제감각 캘리브레이션)
- **'관심 있다고 한 챕터(Q11)' 본인 정답률** vs **'싫다고 한 챕터(Q12)' 본인 정답률** ← 1인 1장 카드의 핵심 인사이트
- 문제유형별(지식축적/이해/적용 등) 정답률

### Phase 3. 진단↔성적 연결 분석 — 전적으로 코드
- **상관 매트릭스**: 진단 약 50개 수치변수 × 시험 점수
- **회귀(LASSO 또는 OLS)**: 어떤 진단 변수가 점수를 가장 잘 예측하는가
- **군집화(K-means k=4–6, 또는 계층)**: 진단 변수 기반 학생 군집화 → 군집별 평균 점수 비교
- **부분군 비교**: 고교생물 이수 여부 / 직업(풀타임/파트타임/무직) / 학력 → 점수 차이 t-test, ANOVA
- **학술 근거**: motivation·learning strategies·anxiety가 학업성취의 강력한 예측변수임이 다수 longitudinal 연구로 입증됨 (paideia-idea.md §3.3 참조)

### Phase 4. 위험군·기회군 자동 라벨링 — 코드 룰엔진 + 미리보기 UX
사전 정의 if-then 라벨(예시):

| 라벨 | 정의 | 운영 의미 |
|---|---|---|
| 🔴 예상 외 부진 | 점수 백분위 ≤ 20 AND 동기 ≥ 5 AND 자습시간 ≥ 5h | 적극 면담 1순위 |
| 🟡 예상된 부진 | 점수 백분위 ≤ 30 AND (동기 ≤ 3 OR 자습 < 5h OR 고교생물 미이수) | 조기 개입군 |
| 🟢 역전 우수 | 점수 백분위 ≥ 70 AND 진단상 위험요인 ≥ 2개 | 성공요인 인터뷰 가치 |
| ⚪ 고불안 학습자 | Q62 불안 키워드 매칭 | 정서적 지원 우선 |
| 🔵 시간 부족형 | 자습 < 5h AND 직업=파트/풀타임 AND 점수 백분위 ≤ 50 | 학습설계 코칭 |

룰 합의만 한 번 하면 매 학기 재사용 가능. **LLM 불필요**.

**OnTask식 "룰 + 미리보기" UX (v0.1 채택)**:
교수자가 라벨 정의를 조정할 때 CLI에 다음을 즉시 출력:
```
$ paideia immersio label-preview --rule "🔴" --course 2026-1-anatomy
🔴 예상 외 부진 — 12명 매칭

샘플 3건 (무작위):
  20260042 (B반)  점수 47/220, 동기=6, 자습=6h
  20260117 (A반)  점수 52/220, 동기=5, 자습=7h
  20260181 (D반)  점수 41/220, 동기=7, 자습=5h
```
룰 변경의 영향을 즉시 검증 가능. 필수 안전장치.

### Phase 5. 자유서술 분석 (Q62 불안, Q63 추가공유) — 두 갈래
- **LLM 가용 시**: 카테고리 분류(암기 부담, 시간 부족, 따라가기 어려움, 시험 불안, 가정 사정 등) + 핵심 인용문 추출
- **LLM 부재 시**: 키워드 사전 한 번만 구축

```
'암기'    → ['외우', '암기', '기억', '용어']
'시간'    → ['시간', '바쁘', '아르바이트', '병행']
'따라가기' → ['따라', '뒤처', '못 따라', '진도']
'불안/심리'→ ['불안', '두렵', '걱정', '자신감']
'기초부족' → ['기초', '처음', '생물', '용어가 어려']
```

정규식 매칭으로 **80% 정도 분류 정확도는 코드만으로 확보**. 응답 다수가 "없습니다"이므로 실제 분류 대상은 50명 안팎.

### Phase 6. 학생 1인 1장 카드 — 코드 + (선택) LLM

**코드 담당**:
- 그래프(백분위 막대, 챕터별 정답률 막대, 진단 레이더), 라벨 표시
- PDF 일괄 생성 (matplotlib + reportlab 또는 python-pptx)

**LLM 담당(선택)**: 카드 하단 "교수자 코칭 제안 3줄"을 자연스러운 문장으로 다듬기

**LLM 부재 시**: 코칭 템플릿 100개를 미리 작성. 룰엔진이 매칭된 템플릿을 그대로 출력.

```
IF 라벨=🔴 AND 약한챕터=신경계 →
  "본인의 학습 노력에 비해 신경계 단원에서 결과가 낮습니다.
   그림 기반 복습자료 활용과 함께 1:1 면담을 권합니다."

IF 라벨=🔵 AND 시각자료선호 ≥ 6 →
  "학습 시간 확보가 우선 과제. 짧은 시각자료 위주 자투리 학습 전략을 추천."
```

학생마다 같은 표현이 나오긴 하나, **운영상 무방하며 일관성 있는 메시지**.

### Phase 7. 후속 운영 분석
- **신청자 명단과 학생 라벨 대조**(코드) — 🔴🟡 라벨 학생 중 얼마나 신청했는가, 안 한 학생은 누구인가
- **자유서술 카테고리 분류** (LLM 또는 키워드사전)
- **신청 사유 군집화** (LLM이 더 정밀, 코드는 빈도표)

## 5. 개발 전략

코드 파이프라인을 먼저 끝까지 닫고, **마지막에 "LLM 옵션 훅"을 두 군데만 끼워넣는** 구조가 안정적.
LLM 호출이 실패해도 폴백 템플릿이 채워지도록 설계하면, 학기 운영 중 어떤 환경에서도 산출이 멈추지 않음.

```
[Bronze] → [Phase 0 정제 → Silver 마스터] → [Phase 2~4 통계·라벨 → Silver 지표] → [Phase 6 → Gold PDF 카드]
                                                                                          │
                                                                                          └─ 옵션 LLM 훅: 코칭 멘트 자연어화
                                                                                                          (실패 시 템플릿 폴백)
```
